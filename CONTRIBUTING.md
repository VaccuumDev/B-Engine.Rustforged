## CONTRIBUTING.md

Thank you for your interest in contributing to **B-Engine.Rustforged**! Your contributions help improve the engine and support the development of **B-World** and other open-world projects. Please follow the guidelines below to ensure a smooth contribution process.

---

### How to Contribute

1. **Reporting Issues**
   - If you encounter bugs or issues while using the engine, please open a new issue on the GitHub repository.
   - Provide a clear title and detailed description, including steps to reproduce the problem.

2. **Feature Suggestions**
   - Have an idea for a new feature? Please share it by creating a new issue labeled "Feature Request."
   - Describe the feature and how you think it would enhance the engine.

3. **Code Contributions**
   - Fork the repository and create a new branch for your feature or fix:
     ```bash
     git checkout -b feature/my-feature
     ```
   - Make your changes and ensure code quality with tests where applicable.
   - Commit your changes with a clear message:
     ```bash
     git commit -m "Add my feature"
     ```
   - Push your changes:
     ```bash
     git push origin feature/my-feature
     ```
   - Create a pull request (PR) to the main repository. Include a description of your changes and reference any related issues.

4. **Code Standards**
   - Write clear, concise code that adheres to existing coding style and conventions used in the project.
   - Include comments where necessary to improve code readability.

5. **Testing Your Changes**
   - Ensure your changes do not break existing functionality by running all tests before submitting a PR.
   - If adding new features, consider writing tests that cover your changes.

6. **Documentation**
   - Update the documentation as needed to reflect your changes.
   - Ensure that any new features are documented clearly in the README or other relevant documentation files.
